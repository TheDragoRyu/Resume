## Problem
Needed a hands-on project to learn Go beyond tutorials — something with real logic, state, and physics to drive it.

## Solution
Built a particle system engine from scratch in Go, modeling the behavior and structure of Unity's Particle System. Particles spawn, move, and expire according to custom rules written entirely without external game frameworks.

## Highlights
- Particle lifecycle management: spawn, update, and expiry handled in pure Go
- Physics-inspired motion modeled after Unity's particle behavior
- Zero external dependencies — all simulation logic hand-rolled
- Demonstrates low-level systems thinking in a new language
